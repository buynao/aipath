提示工程：和 AI 对话的艺术

角色设定、少样本示例、思维链……提示工程不是玄学话术，而是一套有原理可循的工程方法。

基础 约 25 分钟

🎯 你将学会

✓一句话说清 prompt 的本质：给"文字接龙"设定续写条件 —— 从此能自己鉴别任何提示技巧是工程还是玄学
✓掌握五大技法：设角色受众、给示例、给步骤、定格式、划边界 —— 每一招都知道它为什么有效
✓分清 system 和 user 两类提示词：全局约束放哪、为什么放那里更"管用"
✓会写结构化长 prompt（信息分区 + 重点放两头），并能亲手把「帮我写点东西」改造成能直接交付的好 prompt

💡 核心概念：你打出的每个字，都在给 AI 划"续写区"

网上流传着无数"ChatGPT 神级咒语大全"，仿佛和 AI 对话靠的是背口诀。先把底牌亮出来：第 12 课你已经知道，大模型唯一会做的事是接着前文往下接龙 —— 每生成一个 token 之前，它都要对词表里几万个候选算一遍概率（第 14 课）。那么 prompt 是什么？它不是"命令"，而是接龙的前文。模型训练时读过整个互联网，什么样的前文后面通常跟什么样的后文，全被压进了它的参数里 —— 你写下的每一个字，都在改变"接下来最可能出现什么"的概率表。

直觉印象
prompt 是下给 AI 的命令 → 措辞够"讨喜"，它就愿意好好干

按这个理解，提示工程就是揣摩 AI 的"心情"，自然会滑向收集咒语、迷信话术。

真实机制
prompt 是续写的条件 → 每个字都把后文分布推向某片训练数据区

提示工程 = 用文字给模型"圈地"：把它的续写范围从"全网平均"收窄到你想要的那片文本区域。

用这个视角看一个你大概率用过的技巧：

「假装你是一位资深合同律师，帮我审一下这份租房合同。」—— 为什么这一句立刻让回答变专业？
不是模型"入戏了"这种魔法。训练数据里，"资深律师"这个词周围聚集着海量法律文本：合同条款、法律意见书、风险提示。这句话一出，后续 token 的概率分布就对齐到那片区域 —— 措辞变严谨、主动逐条找风险、引用条款编号，全是那片文本区的统计特征。角色扮演有效，原理就这么朴素。

光看例子不过瘾，亲手"圈"一次。同一个问题「解释一下量子计算」，右侧四味配料随意增删，左图实时显示两件事：模型接下来最可能续写成哪种文体，以及续写范围被收得多窄。建议玩法：先全关，看"全网平均"长什么样；再一味一味加上去；最后随手撤掉一味 —— 看分布立刻弹回去。

🎛️ 交互演示 · 续写区"圈地机" 点右侧配料增删 · 左图分布实时联动

把你平时在 ChatGPT / Claude 里见过的现象，和这条机制连上线：

你在对话里看到的现象背后的机制
同一个问题，加一句"你是儿科医生"，答案立刻变专业 角色词把续写分布推向训练数据中"医生文本"的区域
问题写得含糊，答案也跟着又水又泛 含糊前文对应"什么都能接"的宽分布，输出只能取平均 —— 套话是最安全的延续
给了一两个范例，输出风格和格式立刻整齐 范例在上下文里划出一片窄分布，"照着范例续写"是概率最高的路径
用中文问编程问题，回答里总夹着英文术语 训练数据里编程讨论以英文为主，那片区域的统计特征跟着渗了出来

理解到这一层，你手里就有了一把万能检验尺：以后再看到任何"提示词技巧"，先问一句 —— 它有没有把分布推向我想要的区域？讲得通，才值得收藏；讲不通，多半是时灵时不灵的偏方。这把尺子，是本课和满天飞的"话术帖"的根本区别。还有一个细节先埋下：prompt 不只是你刚打的那行字，整个对话历史（包括模型自己之前的回答）都是续写条件 —— 这件事的代价和边界，第 17 课讲上下文窗口时算总账。

📖 五大技法：每一招都讲得通，才招招稳定

这五招覆盖日常九成的场景。每招给你三样东西：一个反例、一个正例、一句"为什么有效"。注意一个共同点：正例里没有任何花哨措辞，全是朴素的信息增量 —— 你是谁、给谁看、长什么样、什么不能做。改变信息而不是改变语气，这正是工程和玄学的分界线。

✗「解释一下量子计算。」
✓「你是科普专栏作者，向完全没学过物理的高中生解释量子计算：多用比喻，不出现公式，800 字以内。」

技法 ① 设定角色与受众。角色定"谁在写"（口吻、用词、知识深度），受众定"写给谁"（详略、比喻多少）。两头都不说，模型只能落在"百科条目式平均答案"上 —— 那是分布最宽、最平庸的位置。一句机制：角色词 + 受众词，把续写分布从"全网均值"收窄到"那类作者写给那类读者"的文本区。

✗「帮商品起名，要高级感但不浮夸、朗朗上口、有记忆点、最好带点东方意境……」（形容词堆到第十个，输出依然各凭运气）
✓「帮商品起名。参考两个我满意的：山茶序（护手霜）、栖云盏（香薰蜡烛）。现在给一款檀香木梳起 3 个名字。」

技法 ② 给示例（few-shot）。与其描述要求，不如给两个范本：形容词人人理解不同，范例没有歧义。模型最强的本事恰恰是模式延续 —— 上文出现几个风格一致的"输入 → 输出"对，续写时它会自动沿用同一套隐含规则，连你说不清的"feel"都能学走。这叫 few-shot / 上下文内学习，是 GPT-3 论文最重要的发现之一，也是预训练接龙能力（第 12 课）的直接红利。

✗「这道行程应用题，直接告诉我答案。」
✓「请一步步思考：先列出已知条件，再写计算过程，最后单独一行给出答案。」

技法 ③ 给步骤（激活思维链）。第 15 课见过多步推理"走钢丝"：一步踩空，满盘皆输。让模型把中间步骤写出来，后面的 token 就是在"已写出的前几步"这个条件下生成的 —— 相当于把心算变笔算，每一步都踩在纸面上。这就是思维链（CoT）。第 23 课的推理模型干脆把"先打草稿"训练成了默认动作，对那类模型这句话可以省了；但对普通模型和复杂任务，它依然是性价比最高的一招。

✗「把这些信息整理成结构化数据给我。」
✓「严格按这个格式输出，不要输出其他内容：{"name": "商品名", "price": 数字, "tags": ["标签"]}」

技法 ④ 定格式（要 JSON 就给 JSON 模板）。"结构化"有一万种长法：表格？列表？字段叫什么？分布太宽，输出就随机。模板是上下文里最强的锚点 —— 续写最顺滑的路径就是照着模板填空。当输出要交给程序处理时（第 19 课工具调用、第 26 课写代码），这招从"加分项"变成"必需品"：格式错一个引号，下游代码就崩。

✗「我们公司 2025 年 Q3 的退货率是多少？」（模型不知道，也会一本正经编一个）
✓「根据我贴的这份报表回答；如果报表里没有这个数字，直接回答"报表中未提供"，不要估算。」

技法 ⑤ 划边界（给"不知道"留一条出路）。训练数据里，流畅自信的下文远多于"中途承认不会" —— 你不开口子，"说不知道"这条路在分布里几乎不存在，模型只能硬编（幻觉的重要来源，第 29 课细讲）。明确许可"承认不确定"，等于把这条出路的概率抬起来，幻觉率显著下降。各家官方提示指南都把这招列为降幻觉的第一线手段。

📖 system 提示词：导演递给演员的人物小传

打开任何一个大模型 API（第 26 课你会亲手调），你会发现发给模型的消息分成三种角色：system、user、assistant。一个比喻就够了：这是一出戏 —— system 是开拍前导演递给演员的人物小传（你是谁、什么性格、哪些事绝不能做），user 是开拍后观众递上来的一句句台词。台词一直在换，人物小传贯穿全场。

幕后 · 整场有效
system：人物小传
放全局行为约束：身份与口吻、做事准则、安全红线、固定的输出格式。先入为主、压在每一次续写上，普通用户通常看不见它。
台前 · 随对话流动
user：台词
放当前这一件事：具体问题、材料、临时要求。每一轮都在变，跟着对话往前滚动。

看一段真实风格的人物小传：

system：「你是'拓智书店'的客服助手。只回答与本店订单、配送、退换货相关的问题；语气友好简洁，每次回答不超过 3 句；查不到订单时引导用户提供订单号，绝不编造物流信息。」
之后用户无论怎么聊 —— 问书、闲聊、套话 —— 这些约束都压在每一次续写上。你在各种产品里见过的"人设"：客服、毒舌影评人、苏格拉底式导师，本质都是一段你看不见的 system prompt。ChatGPT 的"自定义指令"、Claude 的项目设定，填的也是这个字段。

为什么全局约束放 system 比塞进 user 里更"管用"？两个原因。其一是位置：system 永远排在对话最前面，整场不动。其二更关键 —— 训练使然：第 13 课的对齐阶段，模型被专门调教成按"system 要求优先于 user 要求"的层级行事（厂商管这叫指令层级），遇到冲突时倾向守住 system 的规矩。所以同一句"回答不超过 3 句"，写在 user 里聊几轮就可能被冲淡，写在 system 里要稳得多。第 26 课写你的第一个聊天机器人时，system 就是你要填的第一个参数 —— 那一刻你会发现：所谓"开发一个 AI 应用"，一半的功夫就是把这份人物小传写好。

空口无凭，做个对照实验：同一条约束「每次回答不超过 3 句」，一次写进 user 的第一句话，一次写进 system，然后往下多聊几轮 —— 看哪边先失效：

🧱 长 prompt 结构化：把提示词当文档写，而不是当聊天发

任务一复杂，prompt 就会长到几百上千字：背景、要求、材料、范例搅成一锅粥。这时最常见的失败不是"模型不会"，而是"模型没注意到" —— 你明明写了"别超过 100 字"，它就是当没看见。两条结构原则，治的就是这个病：

原则一
信息分区
背景 / 任务 / 约束 / 示例各立门户，用小标题或分隔线隔开。结构清晰的前文对应训练数据里说明书、需求文档那片"高质量后文"区域；分区也让模型生成时更容易"回头对照"相应区块（注意力的拿手活，第 9 课）。附带的好处：逼你自己先想清楚。
原则二
重点放两头
对很长的上下文，模型对中间部分的内容利用得最差 —— 研究者称之为 lost in the middle（第 17 课拆它的机制）。所以关键约束开头定调、结尾重申，别埋在第三段中间等着被"看丢"。

两条原则合起来，一个像样的长 prompt 长这样：

【背景】我们是面向程序员的在线教育公司，下周发布新课《Rust 入门》。
【任务】写 3 条课程预告文案，发在技术社区。
【约束】每条不超过 60 字；语气专业、克制；不出现"家人们"等口水词；不承诺"轻松速成"。
【示例】上次效果最好的一条：「指针让你头疼了十年？Rust 用所有权换你一夜安眠。新课上线，首周 5 折。」
【再次强调】只输出 3 条文案本身，不要解释。

注意最后一行 —— 把最在乎的要求在结尾重申一遍，这就是"重点放两头"。这份模板眼熟吗？它就是把五大技法（角色受众可以补进背景区、示例、格式、边界）装进了一个有秩序的容器里。

🛠️ 实战改造：从「帮我写点东西」到能直接交付

把全课串起来。真实场景：你帮一家咖啡馆运营公众号，想让 AI 写新品推送。看一个烂 prompt 如何四步进化 —— 每一步只用本课讲过的技法，没有一句咒语。

第 0 版 · 碰运气什么条件都没给
「帮我写点东西。」

输出质量：一篇不知道给谁看的通用小作文。分布全开 —— 写什么、给谁、多长，全靠模型猜，它只能交出"全网平均水平"的安全答案。这不是模型不行，是你什么续写条件都没设。

第 1 版 · 技法①加角色与受众
「你是一家社区咖啡馆的主理人。给常来的老顾客写一条微信推送，介绍下周一上新的桂花拿铁。」

输出质量：口吻和对象立住了 —— 开始有"街坊熟客"的亲近感，还会自然带出"秋天第一杯"这类应景表达。但长度忽长忽短、重点忽左忽右：你还没示范"好"长什么样。

第 2 版 · 技法②④加示例与格式
前文 + 「参考这条上次反响最好的推送：『天冷了，焦糖云朵降落本店☁️ 老位置给你留着。』格式：第一行 8 字以内标题，正文不超过 80 字，结尾一个 emoji。」

输出质量：有了范本和模板，文风与结构一次到位，八成可以直接发。剩下两个小毛病：偶尔编造不存在的优惠（"第二杯半价"纯属脑补），而且每次只给一个版本，不合心意就得重抽。

第 3 版 · 技法⑤加边界与评判标准
前文 + 「只提我给的信息，没提的优惠活动不要编。给 3 个备选，每条后面用一句话注明主打角度（怀旧 / 限时 / 口感），方便我选。」

输出质量：「不要编」封掉幻觉出口；「3 个备选 + 注明角度」把挑选成本也交给了模型 —— 从"能用"升级到"好选"。回看全程：从「帮我写点东西」到这里，增加的全是信息：你是谁、给谁看、长什么样、什么不能做。这就是提示工程的全部秘密 —— 模型的能力一直都在，prompt 决定它发挥出几成。

🧪 检验尺上手：工程还是玄学？

开课时发的那把检验尺，现在拿出来用：下面 6 条网上流传的提示技巧，先自己判断 —— 它有没有把分布推向想要的区域？有没有增加真实的信息？想好了再点卡片对答案。

⚠️ 常见误区

✗"给你 100 美元小费""深呼吸再回答"这类网传咒语是提示工程的精髓，照抄就灵
✓能从分布角度讲通的技巧才稳定；讲不通的偏方时灵时不灵，换个模型、换个版本就失效

病因：把个例当规律。这类偏方多半源于某人某次测试的截图：它们或许短暂蹭到了训练数据里某些角落的统计相关性，但效应弱、不同模型表现两极，新一轮训练后随时蒸发 —— 你无法解释它，自然也无法指望它。五大技法之所以跨模型稳定，是因为它们改变的是信息本身（角色、范例、步骤、格式、边界），而不是玄学措辞。拿出那把检验尺：讲不通分布的，别收藏。

✗只要 prompt 写得够好，模型什么问题都能答对
✓prompt 只能引导分布，不能注入模型没有的知识 —— 知识边界要靠 RAG（第 18 课）和工具调用（第 19 课）来补

病因：把"措辞问题"和"知识问题"混为一谈。模型不知道你公司上周的会议结论、今天的股价、内部价格表 —— 这些不在训练数据里，prompt 再精妙，它也只能编。提示工程解决的是"它会，但没发挥好"；RAG 和工具解决的是"它根本不知道"。一个实用的判断法：换十种问法都答错或编造的，不是 prompt 的锅，该去第 18、19 课找答案了。

✍️ 小练习

1. 同事的 prompt 是「帮我总结这份报告」，结果输出又长又泛。请用五大技法至少改三处，并用"分布"的语言说明每一处为什么有效。
查看答案
参考改法：① 加角色与受众「你是给 CEO 写周报的分析师，读者只有 30 秒」—— 把分布从"通用摘要"收窄到"高管摘要"那片简洁、结论先行的文本区；② 定格式「输出 3 条要点，每条不超过 20 字，最后附 1 条风险提示」—— 模板锚点让结构不再随机；③ 划边界「只用报告里出现的信息，报告没提的不要推测」—— 给"不写"留出路，防止模型为凑字数编造。还可以给示例：贴一份以前满意的摘要当范本，让模型做模式延续。
2. 为什么"请一步步思考"能让数学题正确率明显上升？它和第 23 课要讲的推理模型是什么关系？
查看答案
把心算变笔算。多步推理像走钢丝，一步错全盘输；让模型先写出中间步骤，后面的 token 是在"已写出的前几步"这个条件下生成的，每一步都踩在纸面上，错误率自然下降 —— 这就是思维链。与推理模型的关系：思维链原本是用户手动触发的 prompt 技巧；推理模型（o 系列、DeepSeek-R1 等）通过训练把"先打草稿再作答"内化成了默认行为 —— 同一个思想，从"提示技巧"升级成了"模型能力"。
3. 判断：模型总答不对你们公司内部的产品价格表，该继续打磨 prompt，还是该换方案？为什么？
查看答案
换方案。内部价格表不在训练数据里 —— prompt 只能引导模型已有知识的发挥，无法注入新知识，逼问下去只会得到一本正经的编造。正确做法：用 RAG 把价格表检索后放进上下文（第 18 课），或让模型调用查价工具（第 19 课）。检验口诀：换十种问法都治不好的，是知识问题，不是措辞问题。
